# 输出规范

## 目的

作为 `heroagent` 的统一输出规范源，定义所有动作下的固定输出结构、字段顺序和质量要求，保证输出可复用、可检查、可衔接下一步。

## 通用规范

所有动作输出都应满足以下要求：

1. 第一部分直接给结论
2. 第一行单独输出 `[HeroAgent]`
3. 字段名稳定，尽量不要同义改写
4. 缺失信息明确标注，不伪造内容
5. 最后给出 `下一步`

## 通用字段规则

- 所有字段优先使用中文固定字段名
- 同一动作不要混用近义字段
- 若字段无内容，可留空，不要删字段名
- 若需要落盘，优先与 `assets/templates/` 保持一致

推荐通用收尾字段：

- `下一步`：最合理的后续动作

## 动作规范

### `init`

必备字段：

- `工作区`
- `初始化结果`
- `已创建`
- `下一步`

质量要求：

- `初始化结果` 只写 `已初始化`、`已补齐`、`无需变更`
- `已创建` 应列出关键目录或关键文件
- 初始化后必须给出一个明确的后续动作

### `want`

必备字段：

- `目标`
- `背景`
- `价值`
- `范围`
- `不做什么`
- `成功标准`
- `约束`

质量要求：

- `目标` 必须可判断是否达成
- `成功标准` 不能只是“做好”“优化”
- `不做什么` 必须能收边界

### `plan`

必备字段：

- `阶段`
- `产出`
- `风险`
- `完成标志`

质量要求：

- 至少能看出先后顺序
- 每个阶段都要有清晰产出
- `完成标志` 必须可验证

### `todo`

必备字段：

- `任务名`
- `完成定义`
- `依赖`
- `优先级`

质量要求：

- `任务名` 用动作开头
- `完成定义` 要能判定 done
- `依赖` 为空时也可写“无”

### `focus`

必备字段：

- `当前目标`
- `当前阶段`
- `已完成`
- `进行中`
- `阻塞`
- `下一步`

质量要求：

- 不复述无关历史
- `阻塞` 必须具体，不写空泛风险
- `下一步` 只能给一个最优先动作

### `finish`

必备字段：

- `完成任务`
- `完成证据`
- `对目标的推进`
- `新风险或新机会`
- `下一步`

质量要求：

- `完成证据` 不能只有主观判断
- `对目标的推进` 要解释影响，而不是重复任务名

### `achieve`

必备字段：

- `目标`
- `验收结果`
- `证据`
- `剩余缺口`
- `是否归档`
- `下一步`

质量要求：

- `验收结果` 只使用 `已达成`、`部分达成`、`未达成`
- `部分达成` 时必须写清缺口
- `是否归档` 不能脱离证据单独判断

### `abandon`

必备字段：

- `放弃目标`
- `放弃原因`
- `已投入沉没成本`
- `可保留资产`
- `重启条件`
- `下一步`

质量要求：

- `放弃原因` 要落到价值、时机、资源或方向变化
- `可保留资产` 优先写还能复用的内容

### `reflect`

必备字段：

- `现象`
- `影响`
- `根因`
- `当时漏判了什么`
- `下次如何更早发现`
- `下一步`

质量要求：

- `根因` 不能停在表面症状
- `下次如何更早发现` 必须包含观察信号或检查动作

### `realize`

必备字段：

- `原则`
- `触发信号`
- `适用边界`
- `推荐动作`
- `反例`
- `下一步`

质量要求：

- `原则` 应该可迁移
- `适用边界` 要说明何时不该用
- `反例` 用于防止过度泛化

### `master`

必备字段：

- `流程名`
- `适用场景`
- `入口条件`
- `步骤`
- `出口条件`
- `常见失败点`
- `下一步`

质量要求：

- `步骤` 必须有顺序
- `入口条件` 与 `出口条件` 都要可判断
- `常见失败点` 要能指导执行避坑

### `synthesize`

必备字段：

- `教什么`
- `适合对象`
- `核心模型`
- `示例`
- `常见误解`
- `一句话带走`
- `下一步`

质量要求：

- `核心模型` 要足够简洁
- `示例` 优先真实、最小、可复述
- `一句话带走` 要能单独成立

### `forget`

必备字段：

- `淘汰对象`
- `过时原因`
- `失效信号`
- `替代认知或替代流程`
- `迁移建议`
- `下一步`

质量要求：

- `过时原因` 必须说明为什么现在不成立
- `替代认知或替代流程` 不能为空
- `迁移建议` 应可执行，而不是一句“以后不要用了”

## 使用规则

- `SKILL.md` 中涉及输出时，以本文件为准
- `prompt-patterns.md` 只负责提问策略，不重复定义输出字段
- `assets/templates/` 负责落盘模板，应尽量与本文件保持同构
