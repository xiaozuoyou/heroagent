# 输出规范

## 目的

作为 `heroagent` 的统一输出规范源，定义所有动作下的固定输出结构、字段顺序和质量要求，保证输出可复用、可检查、可衔接下一步。

## 通用规范

所有动作输出都应满足以下要求：

1. 第一部分直接给结论
2. 第一行使用单行状态头
3. 主体优先回答固定槽位
4. 字段名稳定，尽量不要同义改写
5. 缺失信息明确标注，不伪造内容
6. 最后给出 `下一步`

## 人类优先规则

- 默认把“人类先看什么”放在最前面，而不是把完整分析放在最前面
- 单次回复优先服务“继续推进”而不是“展示思考过程”
- 若存在需要用户决策的内容，优先压缩成低成本选择，不要要求用户读长文后自行总结
- 若用户没有回应某个选择，默认视为“暂不处理”，不要反复追问

## 统一槽位

除纯问答外，所有正式输出优先固定为以下四段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

若确实需要用户做决定，再补：

- `需要确认`

## 选择格式

当需要用户做决定时，统一优先使用轻量选择格式：

```md
请直接回复：
1. ...
2. ...
3. 暂不处理
```

规则：

- 优先提供 2 到 3 个选项
- 若是二选一，也必须加上 `3. 暂不处理`
- 用户若不回复或避而不谈，视为 `暂不处理`
- 不要把多个不同维度的问题混在同一组选项里

## 顶层输出结构

统一使用以下结构：

```md
{状态符号} HeroAgent · {场景名称}

当前结论：
- ...

当前依据：
- ...

产物或状态更新：
- ...

下一步：
- ...
```

如果需要用户确认，则在 `下一步` 前插入：

```md
需要确认：
- ...
```

## 状态符号

- `✅`：已完成
- `❓`：需要用户输入或确认
- `⚠️`：存在风险、缺口或部分完成
- `🚫`：用户取消或已终止
- `❌`：执行错误
- `💡`：纯问答或轻咨询

## 场景名称

推荐使用：

- 初始化
- 目标定义
- 计划制定
- 任务拆解
- 当前态势
- 目标验收
- 放弃收口
- 问题复盘
- 知识维护
- 咨询问答
- 需要确认
- 执行错误

## 通用字段规则

- 所有字段优先使用中文固定字段名
- 同一动作不要混用近义字段
- 若字段无内容，可留空，不要删字段名
- 若需要落盘，优先与 `assets/templates/` 保持一致
- emoji 只用于状态，不堆砌装饰性符号

## 动作规范

### `init`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前结论` 只写 `已初始化`、`已补齐`、`无需变更`
- `产物或状态更新` 应列出关键目录或关键文件
- 初始化后必须给出一个明确的后续动作

### `wiki`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前依据` 应明确消费了哪些 wiki 或草稿
- `产物或状态更新` 应说明已更新、已读取、已补齐、已生成草稿，或无需变更
- 若本次只生成草稿，必须明确标注“未直接写入正式 wiki”
- 若本次刷新了索引或注册表，也应说明是否同步完成

### `want`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前依据` 至少覆盖评分、边界和约束中的关键项
- 正式进入目标定义前，先完成需求完整度评分
- 评分 `< 8` 时，不输出正式目标卡片，改用澄清输出

### `want` 评分不足时的澄清输出

必备字段：

- `当前结论`
- `当前依据`
- `需要确认`
- `下一步`

质量要求：

- `当前依据` 必须包含当前评分和关键缺口
- 每一轮只允许 1 个最关键问题
- 允许用户回复“按现有信息继续”

### `plan`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `产物或状态更新` 至少能看出阶段顺序、关键产出、风险和完成标志
- 若阶段较多，只在 `当前结论` 里保留关键路径和最近一步

### `todo`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `产物或状态更新` 应包含任务、依赖和优先级
- 默认执行策略是按优先级连续推进全部任务，不逐项请求用户确认
- 只有遇到关键取舍、高风险操作、权限问题或外部阻塞时，才允许中断并请求确认
- `todo` 全部完成后，应自动更新为“已完成，待验收”状态，并将下一步切到 `achieve`

### `focus`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `focus` 是观察动作，不是推进阶段
- `当前结论` 只回答当前最值得关注的内容
- `当前依据` 应指向已完成、进行中、阻塞和项目背景
- `下一步` 只能给一个最优先动作

### `achieve`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前结论` 只使用 `已达成`、`部分达成`、`未达成`
- `当前依据` 必须包含证据和缺口
- 不允许在没有证据时直接归档

### `abandon`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前依据` 要落到价值、时机、资源或方向变化
- `产物或状态更新` 应优先写还能复用的资产和重启条件

### `reflect`

必备字段：

- `当前结论`
- `当前依据`
- `产物或状态更新`
- `下一步`

质量要求：

- `当前依据` 必须区分事实、判断、待确认
- `产物或状态更新` 应覆盖现象、影响、根因、漏判点、提前信号
- 证据不足时，不强行给完整根因，改为列出 `待确认`
