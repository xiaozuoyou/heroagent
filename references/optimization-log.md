# Skill 优化日志

## 2026-03-15

### 目标

收缩 `SKILL.md`，把主文档从“全量操作手册”重构为“高信号入口协议”。

### 命中的问题模式

- `SKILL.md` 同时承担入口说明、动作细则、脚本索引、wiki 策略说明，主线过重
- 与 `README.md`、`references/` 存在重复表达，后续容易漂移
- wiki 能力扩张较快，主文档开始挤压推进闭环的识别信号

### 生效的优化动作

- 保留 `SKILL.md` 中必须稳定存在的内容：适用场景、动作入口、核心规则、输出约定、按需加载入口
- 删除详细动作解释、重复脚本展开和大段执行说明，把细节下放到 `references/`
- 保留脚本分组，但只保留高层索引，不再把脚本列表写成手册

### 指标变化

- `SKILL.md` 行数：`287 -> 170` 左右
- 结构清晰度：`6 -> 8`
- 可执行性：`8 -> 8`
- 可维护性：`5 -> 8`
- 可验证性：`7 -> 7`
- 总分：`6.5 -> 7.75`

注：评分为本次重构后的人工估算，用于后续对比。

### 未解决或待观察问题

- `README.md` 仍保留较多动作示例与脚本清单，后续可能继续与 `SKILL.md` 漂移
- 缺少文档一致性测试，无法自动校验文档引用和脚本列表
- `openai.yaml` 的对外提示语尚未同步最新 wiki 能力定位

### 下次优化建议

- 补一组文档一致性检查，至少覆盖脚本引用、动作入口和关键 references
- 评估是否继续压缩 `README.md`，把它定位为安装与外部使用说明
- 为 `wiki` 的 `conservative`、`aggressive` 策略补齐显式验证
