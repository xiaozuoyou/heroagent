# 提问与输出模板参考

## 目的

为 `heroagent` 的高频动作提供低摩擦提问模板和标准输出模板，降低临场组织成本。

## 使用原则

- 优先用短问题收敛，不一次性追问过多信息
- 提问只补当前动作所需最小信息
- 输出先给结果，再给依据
- 模板是骨架，不是必须逐字照搬

## `want`

### 适用场景

用户表达了想法，但目标尚不清晰。

### 推荐提问

- 你现在最想达成的结果是什么
- 这件事为什么重要
- 你希望做到什么程度算完成
- 这次明确不做什么

### 标准输出模板

```md
## 目标卡片
- 目标：...
- 背景：...
- 价值：...
- 范围：...
- 不做什么：...
- 成功标准：...
- 约束：...
```

## `plan`

### 适用场景

目标已明确，但还没有分阶段路径。

### 推荐提问

- 这件事最晚要在什么时候完成
- 中间有哪些关键节点必须先完成
- 哪一步最容易卡住

### 标准输出模板

```md
## 里程碑计划
1. 阶段：...
   产出：...
   风险：...
   完成标志：...
2. 阶段：...
```

## `todo`

### 适用场景

已有计划，需要落成执行单元。

### 推荐提问

- 现在最先做哪一段最有价值
- 哪些任务存在依赖
- 你希望先保速度还是先保质量

### 标准输出模板

```md
## 任务列表
- [ ] 任务名
  完成定义：...
  依赖：...
  优先级：高 | 中 | 低
```

## `focus`

### 适用场景

用户需要看当前进度、优先级或阻塞。

### 推荐提问

- 你现在最关心进度、风险，还是下一步
- 最近一次明确完成的事项是什么
- 当前最大的阻塞是什么

### 标准输出模板

```md
## 进度快照
- 当前目标：...
- 当前阶段：...
- 已完成：...
- 进行中：...
- 阻塞：...
- 下一步：...
```

## `finish`

### 适用场景

用户反馈某项任务已经完成，需要更新状态。

### 推荐提问

- 这项任务完成的证据是什么
- 它具体推动了哪个目标
- 完成后暴露了什么新风险

### 标准输出模板

```md
## 完成记录
- 完成任务：...
- 完成证据：...
- 对目标的推进：...
- 新风险或新机会：...
- 下一步：...
```

## `achieve`

### 适用场景

需要判断目标是否真正闭环。

### 推荐提问

- 现在有哪些证据能证明目标已达成
- 若说还没完全达成，缺口在哪里
- 这个目标现在适合归档吗

### 标准输出模板

```md
## 达成结论
- 目标：...
- 验收结果：已达成 | 部分达成 | 未达成
- 证据：...
- 剩余缺口：...
- 是否归档：是 | 否
```

## `abandon`

### 适用场景

目标不再值得推进，需要明确停止。

### 推荐提问

- 为什么现在不值得继续做
- 已经投入的东西里什么值得保留
- 未来什么条件下可以重启

### 标准输出模板

```md
## 放弃结论
- 放弃目标：...
- 放弃原因：...
- 已投入沉没成本：...
- 可保留资产：...
- 重启条件：...
```

## `reflect`

### 适用场景

需要复盘问题成因。

### 推荐提问

- 这次最明显的问题现象是什么
- 它造成了什么影响
- 真正的根因是什么，而不是表面症状
- 以后如何更早发现

### 标准输出模板

```md
## 反思记录
- 现象：...
- 影响：...
- 根因：...
- 当时漏判了什么：...
- 下次如何更早发现：...
```

## `realize`

### 适用场景

需要把一次事件提炼成原则。

### 推荐提问

- 这件事暴露出的稳定规律是什么
- 什么信号一出现，就该用这个原则
- 这个原则不适用于哪些情况

### 标准输出模板

```md
## 原则卡片
- 原则：...
- 触发信号：...
- 适用边界：...
- 推荐动作：...
- 反例：...
```

## `master`

### 适用场景

原则已反复验证，需要固化为流程。

### 推荐提问

- 这个做法已经在哪些场景重复验证过
- 进入这套流程的入口条件是什么
- 哪一步最容易失败

### 标准输出模板

```md
## 流程卡片
- 流程名：...
- 适用场景：...
- 入口条件：...
- 步骤：
  1. ...
  2. ...
  3. ...
- 出口条件：...
- 常见失败点：...
```

## `synthesize`

### 适用场景

需要把方法教给别人。

### 推荐提问

- 对方是什么水平，最容易卡在哪里
- 这个知识点最值得记住的核心模型是什么
- 有没有一个最小例子可以直接说明

### 标准输出模板

```md
## 教学稿
- 教什么：...
- 适合对象：...
- 核心模型：...
- 示例：...
- 常见误解：...
- 一句话带走：...
```

## `forget`

### 适用场景

旧知识、旧流程已经失效，需要显式淘汰。

### 推荐提问

- 这个旧做法为什么现在不成立了
- 哪些信号说明它已经过时
- 替代做法是什么
- 如何帮助团队迁移

### 标准输出模板

```md
## 淘汰记录
- 淘汰对象：...
- 过时原因：...
- 失效信号：...
- 替代认知或替代流程：...
- 迁移建议：...
```
